# Uni-Sync 4.0 全局搜索系统实现方案

## 1. 核心定位
**“万能控制台” (Omni-Console)**：不仅仅是查找数据，而是连接用户意图与应用功能的“一键直达”入口。

---

## 2. 搜索覆盖范围 (Search Scope)

### A. 动态业务数据 (SQL Engine)
| 数据类型 | 搜索字段 | 点击行为 |
| :--- | :--- | :--- |
| **待办事项** | 标题、备注、分类名 | 跳转至任务编辑/详情页 |
| **日程课表** | 课程名、教师、教室 | 跳转至周课表并高亮该项 |
| **倒计时** | 标题、目标日期 | 跳转至倒计时详情 |
| **专注日志** | 日志标题、关联任务名 | 跳转至历史记录统计 |
| **日期搜索** | "今天", "昨天", "04/24" 等 | 过滤并展示对应日期的所有记录 (待办/番茄/日志) |
| **应用用时** | 应用名称 | 跳转至该应用的详细使用趋势页 |

### B. 静态设置项 (Settings Registry)
| 类别 | 包含项 (每一个细节) | 点击行为 |
| :--- | :--- | :--- |
| **个性化** | 壁纸、主题、动效、模糊开关 | 跳转至对应设置页并高亮该行 |
| **同步** | 节点选择、同步间隔、全量同步 | 弹出同步配置或执行同步 |
| **系统** | 权限、缓存、迁移、关于 | 直接进入二级设置菜单 |

### C. 快捷动作 (Quick Actions)
| 触发词 | 动作内容 |
| :--- | :--- |
| "新建" | 唤起新建任务弹窗 |
| "开始" | 启动默认番茄钟 |
| "清空" | 触发回收站清理 |

---

## 3. 技术路线图 (Implementation Path)

### 第一阶段：底层模型与服务架构 (✅ 已完成)
1. **SearchResult 模型**：定义统一的结果数据结构。
2. **SearchService 抽象**：`SqlSearchProvider`, `SettingsSearchProvider`。

### 第二阶段：SQL 性能优化与多维检索 (✅ 已完成)
1. **扩展 FTS4 索引**：为核心表建立 FTS 虚表。
2. **日期解析引擎**：集成自然语言日期识别，支持“今天”、“上周二”等语义搜索。
3. **跨表聚合**：单次查询同时检索待办、课表、倒计时及专注日志。

### 第三阶段：Spotlight 交互 UI 与深层跳转 (✅ 已完成)
1. **毛玻璃层 (Blur Overlay)**：精致的交互反馈。
2. **深层路由 (Deep Linking)**：点击搜索结果直接精准导航至二级/三级页面（如屏幕时间详情、特定日期的日志统计）。

---

## 4. 最终视觉与体验效果 (Final Effect)

- **高级感**：搜索框带有精致的微动画，背景实时模糊，适配深色/浅色模式。
- **全域感**：从“查事情”到“搜时间”再到“跳设置”，实现全应用功能的零门槛触达。
- **直觉化**：输入日期即刻汇总当日成果。
- **低延迟**：得益于 SQLite FTS 技术，毫秒级响应。
