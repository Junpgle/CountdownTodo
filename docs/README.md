# Project Documents

This directory collects project documents that used to live in the repository
root. Keep the root for build/config entry points only.

## Structure

- `ai/`: AI agent and ML planning notes.
- `archive/`: Older roadmap and draft planning documents kept for reference.
- `design/`: Design drafts and visual source files.
- `features/`: Feature design and implementation guides.
- `private/`: Sensitive local-only notes, such as test account documents.
- `reports/`: Debugging, migration, and fix reports.
- `sync/`: Uni-Sync, conflict handling, and collaboration design notes.

## Root Exceptions

The following documents intentionally remain in the root:

- `Readme.md`: repository overview.
- `AGENTS.md`: coding-agent repository instructions.
- `CLAUDE.md`: Claude-specific project instructions.
- `PRIVACY_POLICY.md`: app privacy policy.
