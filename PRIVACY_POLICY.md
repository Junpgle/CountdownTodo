# CountDownTodo 隐私政策

**版本日期：2026年4月11日**
**生效日期：2026年4月11日**

CountDownTodo（以下简称"本应用"）由 [Junpgle]（以下简称"我们"）开发并运营。我们深知个人信息对您的重要性，并会尽全力保护您的个人信息安全。本隐私政策旨在帮助您了解我们如何收集、使用、存储和保护您的个人信息。

**请您在使用本应用前，仔细阅读并充分理解本隐私政策。一旦您使用或继续使用本应用，即表示您同意我们按照本隐私政策处理您的个人信息。**

---

## 一、我们如何收集和使用您的个人信息

### 1.1 账号注册与登录

当您注册本应用账号时，我们会收集您的**手机号码或电子邮箱**用于创建账号和身份验证。我们采用自建JWT Token认证机制，您的密码将以加密形式存储在我们的服务器上。

### 1.2 核心功能数据

为提供待办事项管理、倒计时、番茄钟、课表管理等核心功能，我们会收集并存储以下数据：

- **待办事项数据**：您创建的待办事项内容、完成状态、提醒时间等
- **倒计时数据**：您设置的倒计时目标、时间等
- **番茄钟数据**：您的番茄钟使用记录、标签、设置等
- **课表数据**：您导入或创建的课程表信息
- **屏幕使用时间**：各应用的使用时长统计（需您授权"使用情况访问权限"）

### 1.3 设备信息

为保障应用正常运行和提供版本更新服务，我们可能会收集以下设备信息：

- 设备型号、操作系统版本
- 应用版本号
- 设备唯一标识符（用于数据同步和在线统计）

### 1.4 多平台支持

本应用支持以下平台，功能可能因平台而异：

| 平台                  | 支持情况 | 特有功能                        |
|---------------------|------|-----------------------------|
| **Android 手机端**     | 完整支持 | 桌面小部件、小米手环通信、屏幕时间统计         |
| **Windows 电脑端**     | 完整支持 | 灵动岛/悬浮窗、多窗口                 |
| **Windows Lite电脑端** | 基础支持 | 悬浮窗                         |
| **vela Band 手环端**   | 基础支持 | 与手机通信                       |
| **网页端**             | 基础支持 | 仅通过 Cloudflare 服务器访问，功能可能受限 |

**特别说明：**
- **灵动岛/悬浮窗功能**：仅 Windows 电脑端可用
- **实时活动通知功能**：需要 Android 16.1及以上版本支持，澎湃OS升级到3.0.301及以上版本支持
- **番茄钟跨端感知功能**：依赖阿里云服务器，手机端和电脑端均可使用
- **网页端**：仅支持通过 Cloudflare 服务器访问，部分功能可能不可用

### 1.5 权限调用说明

本应用可能请求以下系统权限，您可以随时在系统设置中关闭：

| 权限 | 用途 | 是否必需 |
|------|------|----------|
| 网络权限 | 数据同步、版本检查 | 是 |
| 通知权限 | 提醒通知 | 否 |
| 存储权限 | 导入/导出文件、图片识别 | 否 |
| 开机自启 | 重启后恢复提醒 | 否 |
| 精确闹钟 | 准时触发提醒 | 否 |
| 使用情况访问 | 屏幕时间统计 | 否 |
| 悬浮窗权限 | 灵动岛/悬浮窗功能（仅电脑端） | 否 |

### 1.6 AI 功能

当您使用"自然语言创建待办"或"图片识别待办"功能时：

- 您输入的文本或上传的图片将发送至**智谱AI（BigModel）** API进行处理
- 默认使用智谱AI服务，您也可在设置中自定义AI服务商
- 我们不会将您的数据用于AI模型训练
- 如您使用自定义AI服务，数据将发送至您指定的服务器

### 1.7 壁纸服务

当您使用首页壁纸功能时：

- **必应每日一图 (Bing)**：我们会通过 `https://bing.biturl.top` 接口获取必应每日一图及其版权描述。我们会向该接口发送您的地区（mkt）、选择的分辨率等参数，但不会发送任何识别您个人身份的信息。
- **GitHub 随机壁纸**：我们会访问特定的开源仓库获取图片链接。

## 二、我们如何存储您的个人信息

### 2.1 存储方式

我们采用**本地+云端混合存储**方式：

- **本地存储**：您的数据首先保存在设备本地（使用SharedPreferences和SQLite数据库）
- **云端存储**：当您登录账号后，数据将同步至我们的服务器以实现多设备同步

### 2.2 服务器架构说明

本应用使用双服务器架构，各自用途如下：

| 服务器 | 提供商 | 用途 | 传输协议 |
|--------|--------|------|----------|
| **阿里云服务器** | 阿里云（中国大陆） | 番茄钟跨端感知、数据同步、用户认证、排行榜、在线统计等核心功能 | **HTTP（未加密）** |
| **Cloudflare 服务器** | Cloudflare（全球 CDN） | 网页端访问、静态资源配置、版本更新检查 | HTTPS（加密） |
| **GitHub Raw** | GitHub | 开屏页面配置、壁纸配置等静态资源 | HTTPS（加密） |
| **Wallpaper API** | biturl.top | 获取必应每日一图数据 | HTTPS（加密） |

**平台访问说明：**
- **手机端（Android）**：可访问阿里云服务器和 Cloudflare 服务器
- **电脑端（Windows）**：可访问阿里云服务器和 Cloudflare 服务器
- **网页端**：仅可访问 Cloudflare 服务器

### 2.3 存储地点

- 本地数据：存储在您的设备中
- 云端数据：存储于阿里云服务器（中国大陆）和 Cloudflare 服务器
- 静态资源：部分配置信息存储于GitHub Raw

### 2.4 存储期限

- 账号数据：在您注销账号前持续保存
- 本地数据：在您卸载应用前持续保存
- 日志数据：服务器日志保留不超过6个月

## 三、我们如何共享、转让、公开披露您的个人信息

### 3.1 共享

**我们不会主动将您的个人信息共享给第三方。** 以下情况除外：

- **智谱AI**：仅当您使用AI功能时，将您输入的文本或图片发送至智谱AI API进行处理
- **小米穿戴SDK**：仅当您使用小米手环功能时，将待办、课表等同步数据发送至小米穿戴设备
- **法律法规要求**：根据法律法规、诉讼争议解决需要，或按行政、司法机关依法提出的要求

### 3.2 转让

我们不会将您的个人信息转让给任何公司、组织和个人，但以下情况除外：

- 事先获得您的明确同意
- 在涉及合并、收购或破产清算时，如涉及个人信息转让，我们会要求新的持有方继续受本政策约束

### 3.3 公开披露

我们仅会在以下情况下公开披露您的个人信息：

- 获得您的明确同意后
- 基于法律法规、法律程序、诉讼或政府主管部门强制性要求

## 四、我们如何保护您的个人信息

### 4.1 安全措施

我们已采取相应的安全防护措施保护您的个人信息：

- **数据加密存储**：敏感数据在服务器端加密存储
- **访问控制**：严格的服务器访问权限管理
- **HTTPS 加密传输**：Cloudflare 服务器和 GitHub Raw 使用 HTTPS/TLS 加密传输

**重要安全提示：**

**阿里云服务器当前使用 HTTP 协议传输数据（未加密），存在数据被第三方截获的风险。** 我们正在推进服务器备案工作，备案完成后将启用 HTTPS 加密传输。在备案完成前，请您知悉并理解此风险。

### 4.2 安全事件处置

如发生个人信息安全事件，我们将按照法律法规要求，及时向您告知：

- 安全事件的基本情况和可能的影响
- 我们已采取或将要采取的处置措施
- 您可自主防范和降低风险的建议
- 对您的补救措施

### 5.1 访问和更正

您有权访问和更正您的个人信息：

- **账号信息**：在"设置"中查看和修改
- **业务数据**：在相应功能页面中查看、编辑和删除

### 5.2 删除

在以下情形下，您可以向我们提出删除个人信息的请求：

- 我们处理个人信息的行为违反法律法规
- 我们收集、使用您的个人信息，却未征得您的同意
- 我们处理个人信息的行为违反了与您的约定
- 您注销了账号
- 我们终止服务及运营

### 5.3 注销账号

您可以通过以下方式注销账号：

1. 打开应用，进入"设置" > "账号安全" > "注销账号"
2. 联系客服申请注销

账号注销后，我们将：

- 停止为您提供产品或服务
- 删除您的个人信息或进行匿名化处理
- 法律法规另有规定的除外

### 5.4 撤回同意

您可以通过以下方式撤回授权：

- **系统权限**：在设备系统设置中关闭相应权限
- **数据同步**：在应用设置中退出登录或关闭同步功能

撤回同意不影响此前基于您授权已进行的个人信息处理活动的效力。

### 5.5 获取个人信息副本

您有权获取您的个人信息副本。如您需要，可通过客服联系我们获取。

## 十、定义

- **个人信息**：以电子或者其他方式记录的与已识别或者可识别的自然人有关的各种信息，不包括匿名化处理后的信息
- **敏感个人信息**：一旦泄露或者非法使用，容易导致自然人的人格尊严受到侵害或者人身、财产安全受到危害的个人信息
- **匿名化**：通过对个人信息的技术处理，使得个人信息主体无法被识别，且处理后不能被复原的过程
- **设备信息**：包括设备型号、操作系统版本、设备标识符等
